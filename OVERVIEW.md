## 🚀 Quick Start

### Prerequisites

- [Node.js](https://nodejs.org/) (LTS recommended)
- [Java](https://adoptium.net/) 11+ (required by shadow-cljs / ClojureScript compiler)

### Install dependencies

```bash
bun install
```

### Development (hot-reload)

```bash
bun run dev
# open http://localhost:3000
```

shadow-cljs will recompile on every save and hot-reload the browser automatically.

### Production build

```bash
bun run build
# output → docs/js/main.js  (minified, advanced optimisations)
```

### Create a new post stub

```bash
cd docs
./new-post.sh 2026-03-07
# copy the printed stub into src/blog/data.cljs
```

## 📝 Adding a New Post

All posts live as plain Clojure maps inside the `posts` vector in
`src/blog/data.cljs`. There is no separate HTML file to create.

### Minimal post map

```clojure
{:id         "2026-03-07"          ; YYYY-MM-DD, used as the URL key
 :title      "My Post Title"
 :date-range "March 1-7, 2026"
 :month-year "March 2026"          ; shown in the post footer
 :excerpt    "Short blurb for the index card (2-3 sentences)."
 :content
 [[:p "Opening paragraph..."]
  [:h2 "Section"]
  [:p "Body text."]]}
```

### Content is hiccup

`:content` is a vector of [Reagent hiccup](https://reagent-project.github.io/) vectors
that get spliced directly into `<article class="blog-post">`.

Available elements and helpers:

| Hiccup | Renders as |
|---|---|
| `[:h2 "..."]` | Blue section header |
| `[:h3 "..."]` | Teal subsection header |
| `[:p "..."]` | Body paragraph |
| `[:ul [:li ...]]` | Bulleted list (mauve markers) |
| `[:code "..."]` | Inline peach monospace |
| `[:strong "..."]` | Peach bold |
| `[:em "..."]` | Lavender italic |
| `[:div.highlight-box [:p ...]]` | Yellow left-border insight box |
| `[:div.dream-box [:p ...]]` | Mauve left-border dream/reflection box |

### Post ordering

`data/all-posts` sorts by `:id` descending — newest first — so you can
append new maps anywhere in the `posts` vector and ordering is automatic.

## 🗂️ Source File Reference

### `src/blog/core.cljs`

Entry point. Exports `init` (called on first load) and `reload` (called by
shadow-cljs after each hot-reload). Mounts the root `app` component into
`#app`.

### `src/blog/router.cljs`

Tiny hash router. Maintains a single Reagent atom `current-route` of shape
`{:page :index}` or `{:page :post :id "YYYY-MM-DD"}`. Listens to the
`hashchange` event to keep the atom in sync.

Public API:
- `(router/init!)` — wire up the event listener, parse the initial hash
- `(router/navigate-to-index!)` — go to `#/`
- `(router/navigate-to-post! id)` — go to `#/posts/<id>`

### `src/blog/views.cljs`

Two page components:
- `(views/index-page)` — renders the post-card list
- `(views/post-page id)` — renders a single post article

### `src/blog/data.cljs`

All post content. Public API:
- `(data/all-posts)` — sorted vector of all post maps (newest first)
- `(data/get-post id)` — returns one post map or `nil`

### `docs/css/style.css`

Single stylesheet. All Catppuccin custom properties defined in `:root`.
No per-page overrides needed — the same file covers both views.

### `docs/index.html`

Six-line HTML shell. Just sets the charset/viewport, loads
`css/style.css`, provides `<div id="app">`, and loads `js/main.js`.
No content lives here.

## 🔄 Workflow

### Weekly Blog Post Cycle

1. **Collect** — Journal entries throughout the week (`../journals/`)
2. **Review** — Read journal files for the past 7 days
3. **Scaffold** — Run `./new-post.sh YYYY-MM-DD` to get the data stub
4. **Synthesize** — Group by theme, create narrative
5. **Write** — Fill the hiccup content map in `data.cljs`
6. **Build** — `bun run build`
7. **Commit & push** — GitHub Pages serves the updated `docs/`

## 🌐 Deployment

### GitHub Pages Setup

1. Push the repository (ensure `docs/js/main.js` is committed, or let CI build it)
2. Repository → Settings → Pages
3. Source: **Deploy from a branch**
4. Branch: `main` / Folder: `/docs`
5. Save — site is live at `https://username.github.io/repo-name/`

> **Tip**: Add a GitHub Actions workflow to run
> `bun run build` on push so you never need to commit
> compiled JS manually.
